Redis空间引擎设计

发表于 2017-07-15 | 分类于 数据库
字数统计

前言

Redis从3.2版本开始对GEO功能进行了支持，这个功能可以将用户给定的点地理位置信息储存起来， 并对这些信息进行操作。GEO背后原理是基于Geohash，利用Sorted set（有序集合）实现的。

GeoHash是一种地址编码，通过切分地图区域为小方块（切分次数越多，精度越高），它能把二维的经纬度编码成一维的字符串。也就是说，理论上geohash字符串表示的并不是一个点，而是一个矩形区域，只要矩形区域足够小，达到所需精度即可。

Redis内部使用有序集合来保存key，每一个member的score大小为Geohash值。这种使用Geohash作为地理要素索引方案存在四个缺点：

  1. 由于Geohash编码方式，靠近方块边界两侧的点虽然空间距离相隔较近，但编码却完全不同。实际搜索时候不仅要用搜索点的Geohash编码，也要利用环绕搜索点周围八个区域的Geohash值。
  2. Geohash索引能索引基于经纬度的球面坐标，但对平面坐标却无能为力。并且对于小范围的局部搜索，依旧使用耗时的球面计算，性能较低。
  3. Redis中Geohash索引方案只能支持点坐标，而无法索引线、面地理数据。
  4. 不支持常见的INTERSECT|WITHIN搜索。

因此，为了改进Redis存储组织查询空间数据能力，从两方面入手，第一，利用Redis c++ driver，从逻辑上自上而下设计分层数据模型，组织成可调用API形式。第二，针对Redis源码进行了改进，使其支持R树索引。我主要负责第二部分，对Redis源码进行改进。

基于Redis空间数据组织

空间数据库一般分为三个层次：空间数据库(SpatialDatabase)、要素集(FeatureClass)、要素(Feature)。空间数据库、要素集由相应的元数据描述，要素包含几何信息和属性信息。空间数据分层示意图如下图所示：

这些层次的信息可以利用Redis内置结构例如set，hash实现。要素的属性信息（例如线的长度）索引还可以Secondary indexing with Redis这篇文章中提到方法来实现。但是要素几何信息索引就很棘手了，有一种通过Sorted set来建立要素空间范围索引，根据每个要素的MBR(最小包围矩形)的四个坐标数据，分别对四个坐标建立四个关联的Sorted set结构,每个Sorted set的score为要素集所有要素MBR的四个坐标中对应的一个，member为要素ID。以一个简单的例子进行说明。

如上图，我们建立四个key分别为C:MinX,C:MinY,C:MaxX,C:MaxY的四个Sorted set。我们先根据要素ID集合遍历要素集的所有要素，以每个要素对应的MBR的某一坐标为score，要素ID为对应的集合成员，向Sorted Set中添加数据。在进行MBR查询时，分别以待查MBR的四个坐标（假设为minx=20,miny=20,maxx=40,maxy=40）为条件在四个Sorted set中进行包含查询，查询在被待查MBR包含的所有要素。对于C:MinX，使用zrangebyscore C:MinX 20 40对于C:MinY, zrangebyscore C:MinY 20 40对于C:MaxX，使用zrangebyscore C:MaxX 20 40对于C:MaxY, zrangebyscore C:MaxY 20 40。然后将四个查询的结果进行求交，最终得到查询的结果要素的ID为2和3。

这种方法实现起来复杂，维护麻烦，查询效率也不高。所以想要对Redis源码进行改进，原生支持存储点、线、面等OGC标准支持的空间要素，并有R树索引，能够高效进行WITHIN|INTERSECT操作。

Redis的GIS扩展

Redis中有五种对象：字符串、列表、集合、有序集合、哈希。没有一种原生对象能够满足我们的要求，于是我们设计了一种新的Spatial对象，将标准hash结构和R树索引结合在一起。和数据库其它的对象不同，通过特定以G为前缀的命令来维护Spatial对象。新的命令参考了Redis hash结构设计：GSET, GSETNX, GGET, GMSET, GMGET, GDEL, GLEN, GSTRLEN, GKEYS, GVALS, GGETALL, GEXISTS, GSCAN，GSEARCH。前13个命令含义和hash命令含义一样，后面GSEARCH是新的命令，用于特定的地理搜索操作，也就是WITHIN|INTERSECT。所有空间数据在RDB和内存中都以WKB二进制形式存储。AOF重写利用GMSET来操作二进制数据。

移植新命令

为了维护新的Spatial对象，我们为Redis添加了十四个新命令。因为Spatial对象中spatial结构体里除了R树结构其它都是hash结构，所以新的命令参考hash指令，将针对hash指令改为针对Spatial对象。除了十三个由hash命令改过来的，还添加了利用R树进行搜索的GSEARCH指令。这一节分为两个部分，第一部分介绍仿hash命令修改Redis源码过程，第二部分着重介绍GSEARCH命令。

GSEARCH命令实现解析

GSEARCH命令用法

GSEARCH 命令有不同的查询模式和丰富返回结果：

GSEARCH参数含义

  • WITHIN: 完全在target指定范围里的对象会返回。
  • INTERSECTS: 和target指定范围相交对象会返回。
  • CURSOR:利用cursor来分批返回大量数据。目前还未实现CURSOR，现在只是默认全部返回。
  • MATCH:用设定模式筛选返回数据的filed值。

输出形式

  • OUTPUT COUNT: 返回结果数目。
  • OUTPUT FIELD: 只返回filed值，忽略value值。
  • OUTPUT WKT: 返回WKT形式结果，这是默认参数。
  • OUTPUT WKB: 返回WKB形式结果。
  • OUTPUT POINT: 返回结果为XY (经度, 纬度) 点串。
  • OUTPUT BOUNDS: 返回结果为最小包围矩形 (MinX, MinY, MaxX, MaxY)。

目标

  • RADIUS: 搜索范围是一个圆圈，也就是邻近搜索。 需要中心点和半径（单位米）。
  • GEOMETRY: 搜索范围是几何图形，格式可以为WKT|WKB。
  • BOUNDS: 搜索范围为矩形包围盒。 需要minlon, minlat, maxlon, maxlat。
  • MEMBER: 搜索范围是已在数据存在的一个成员。
GSEARCH原理

gsearch函数主要分三个步骤：

  1. 解析gsearch参数，确定searchContext中内容。

第一个步骤没有什么技巧，没什么值得说的。第二个步骤有一些技巧。说说第二个怎么做的。

查询存在两个阶段：首先进行R树索引筛选，筛选出候选集，然后对后选集进行遍历求精。第一阶段的R树搜索是自顶向下搜索，是个递归的过程。看每个节点MBR范围和目标MBR范围有没有overlap。一直递归搜索到R树的叶子节点。这时候开始第二阶段遍历求精，我们在R树搜索函数中会传入iterator函数指针，这时候执行iterator函数，进行遍历求精，填充返回结果。下面是代码逻辑：

RDB和AOF支持

为了支持两种持久化操作，分别在RDB和AOF文件逻辑中加入处理代码。

RDB持久化

OBJ_SPATIAL对象需要持久化的是spatial结构体中的存储空间数据的哈希结构。所以用哈希结构保存函数即可。主要逻辑加入在/src/rdb.c中，主要是save和load逻辑。

  1. save:添加了RDB_TYPE_SPATIAL_ZIPLIST和RDB_TYPE_SPATIAL类型，save逻辑加入在rdbSaveObjectType,rdbSaveObject两个函数中。
  2. load:load逻辑添加在rdbLoadObject函数中，我们从系统中得到还原出来的hash结构，重新遍历这个hash结构，重构OBJ_SPATIAL对象。

AOF持久化

aof部分主要逻辑在重写指令部分，添加了重写OBJ_SPATIAL对象函数,利用了GMSET命令。主要逻辑加入在/src/aof.c中，主要是rewriteAppendOnlyFile函数中，然后模仿rewriteHashObject函数加入了rewriteSpatialObject函数。

Union-Find学习笔记

发表于 2017-05-15 | 分类于 算法学习

Dynamic Connectivity

Given a set of N objects.

  • Union command: connect two objects.
  • Find/connected query: is there a path connecting the two objects?

Modeling the objects

Applications involve Union-Find.

  • Pixels in a digital photo.
  • Computers in a network.
  • Friends in a social network.
  • Transistor in a computer chip.
  • Elements in a mathematical set.
  • Variable names in Fortran program.
  • Metallic sites in a composite system.

Modeling the connections

Assume “is connected to” is an equivalence relation:

  • Reflexive(自反关系): p is connected to p.
  • Symmetric(对称关系): if p is connected to q, then q is connected to p.
  • Transitive(传递关系): if p is connected to q and q is connected to r, then p is connected to r.

Connected components: Maximal set of objects that are mutually connected.

Implementing the operations

Find query. Check if two objects are in the same component.

Union command. Replace components containing two objects with their union.

Union-find data type(API)

Quick-find [eager approach]i

Data structure.

  • Integer array id [] of length N.
  • Interpretation: p and q are connected if and only if they have same id.

Find. Check if p and q have the same id.

Union. To merge components containing p and q, change all entries whose id equals id[p] to id[q]

Quick-find is too slow

Cost model. Number of array accesses (for read or write).

algorithm initialize union find
quick-find N N 1

Union is too expensive. It takes $N^2$ array accesses to process a sequence of N union commands on N objects.

Quick-union [lazy approach]

Data structure.

  • Integer array id [] of length N.
  • Interpretation: id[i] is parent of i.
  • Root of i is id[id[id[…id[i]…]]].

Find. Check if p and q have the same root.

Union. To merge components containing p and q, set the id of p’s root to the id of q’s root.

Quick-union is also too slow

Cost model. Number of array accesses (for read or write,worst case).

algorithm initialize union find
quick-union N N N

Quick-find defect.

  • Union too expensive(N array accesses).
  • Trees are flat, but too expensive to keep them flat.

Quick-union defect.

  • Trees can get tall.
  • Find too expensive(could be N array accesses).

Improvement 1: weighting

Weighted quick-union.

  • Modify quick-union to avoid tall trees.
  • Keep track of size each tree(number of objects).
  • Balance by linking root of smaller tree to root of larger( reasonable alternatives: union by height or “rank”) tree.

Weighted Quick-union Implementation

Data structure. Same as quick-union, but maintain extra array sz[i] to count number of objects in the tree rooted at i.

Find. Identical to quick-union.

Union. Modify quick-union to:

  • Link root of smaller tree to root of larger tree.
  • Update the sz[] array.

Weighted quick-union analysis

Running time.

  • Find: takes time proportional to depth of p and q.
  • Union: takes constant time, given roots.

Proposition. Depth of any node x is at most lgN.

Prove: The depth of node x increases by 1 when tree T1 containing x is merged into another tree T2. The size of the tree containing x at least doubles since T2>=T1. Due to the fact that size of node x is N at most, Size of tree containing x can double at most lgN times.(合并时候深度增加1，个数至少增加一倍，一棵树至多有N个节点，所以树的高度至多为lgN)

Cost model. Number of array accesses (for read or write,worst case).

algorithm initialize union find
weighted QU N lgN lgN

Improvement 2: path compression

Quick union with path compression. Just after computing the root of p, set the id of each examined node to point to that root.

Path compression Implementation

Two-pass implementation: add second loop() to root() to set the id[] of each examined node to the root.

Simpler one-pass variant: Make every other node in path point to its grandparent(thereby halving path length).

Weighted quick-union with path compression: amortized analysis(摊还分析)

Proposition. Stating from an empty data structure, any sequence of M union-find ops on N objects makes<= c(N+Mlg N) array accesses.(lg N为迭代对数函数)

In practice, WQUPC is linear.

Summary

Union-find applications

  • Percolation(蒙罗卡特模拟，渗滤).
  • Games(Go),Hex)).
  • Dynamic connectivity.
  • Least common ancestor(最小公共祖先).
  • Equivalence of finite state automata(有限自动机).
  • Hoshen-Kopelman algorithm in physics.
  • Hinley-Milner polymorphic type inference.
  • Kruskal’s minimum spanning tree algorithm(最小生成树).
  • Compiling equivalence statements in Fortran.
  • Morphological attribute openings and closings.
  • Matlab’s bwlabel() function in image processing.
