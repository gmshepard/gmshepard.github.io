数据库索引简介

发表于 2017-03-02 | 分类于 数据库

数据库中往往存储了海量的数据。面对如此庞大的数据集，如果不为它们建立索引，数据库系统只能使用逐数据扫描的方式去寻找目标记录，这种方式极其低效耗时，显然这对于用户来说是不可接受的。索引通过特殊的数据结构维护了数据在磁盘中的位置信息，通过增加数据库的写代价大大提升了数据库的读效率，已经成为数据库系统中不可或缺的组成部分。索引功能一般由数据库系统的存储引擎提供。

Redis中的数据结构

发表于 2017-02-03 | 分类于 数据库

Redis中使用对象来表示数据库中的键和值，每次当我们在Redis的数据库中新创建一个键值对时，我们至少会创建两个对象，一个对象作为键值对的键，一个作为值。对于键来说，总是一个字符串对象，而值就不一定了。用type命令可以查看当前值的数据结构，它们分别是：string（字符串）、hash（哈希）、 list（列表）、set（集合）、zset（有序集合）。但这些只是Redis对外的数据结构。实际上每种数据结构都有自己的底层的内部编码实现，而且是多种实现，这样Redis会在合适的场景选择合适的内部编码。

I/O多路复用原理

发表于 2017-01-15 | 分类于 计算机网络

I/O多路复用简单来说就是监听感兴趣的 I/O 事件，例如读事件，写事件等，同时也要维护一个以文件描述符为主键，数据为某个预设函数的事件表，这里其实就是一个数组或者链表。当事件触发时，比如某个文件描述符可读，系统会返回文件描述符值，用这个值在事件表中找到相应的数据项（包括回调函数等），从而实现回调。同样的，定时事件 也是可以实现的，因为系统提供的 I/O 多路复用技术中的函数允许我们设置等待超时的时间，预设定时间内没有事件发生时，会返回。
